Conceptos Clave de JavaScript

Variables y Tipos de Datos

JavaScript es un lenguaje de tipado dinámico, lo que significa que las variables pueden cambiar de tipo durante la ejecución del programa.

Tipos Primitivos:

  • String: Para texto y caracteres
  • Number: Para números enteros y decimales
  • Boolean: Para valores verdadero/falso
  • Undefined: Para variables declaradas sin valor
  • Null: Para representar la ausencia intencional de valor
  • Symbol: Para crear identificadores únicos
  • BigInt: Para números enteros muy grandes

Variables Modernas:

  • let: Para variables que pueden cambiar
  • const: Para valores que no deben cambiar
  • var: Forma antigua, evitar su uso

Funciones

Las funciones en JavaScript son objetos de primera clase, lo que significa que pueden ser:

  • Asignadas a variables
  • Pasadas como argumentos
  • Retornadas por otras funciones
  • Almacenadas en objetos

Tipos de Funciones:

  • Funciones Declarativas: Definidas con la palabra clave function
  • Funciones Expresivas: Asignadas a variables
  • Arrow Functions: Sintaxis moderna y más concisa
  • Funciones Constructoras: Para crear objetos

Asincronía en JavaScript

JavaScript maneja la asincronía de varias formas para realizar operaciones que toman tiempo sin bloquear la ejecución:

Callbacks:

  • Funciones que se ejecutan cuando otra operación termina
  • Forma tradicional de manejar asincronía
  • Puede llevar a "callback hell"

Promises:

  • Representan un valor futuro
  • Estados: pending, fulfilled, rejected
  • Permiten encadenar operaciones
  • Mejor manejo de errores

Async/Await:

  • Sintaxis más limpia para promesas
  • Hace que el código asíncrono parezca síncrono
  • Mejor manejo de errores con try/catch

Event Loop y Concurrencia

El Event Loop es el mecanismo que permite a JavaScript manejar operaciones concurrentes con un solo hilo de ejecución.

Componentes Principales:

  • Call Stack: Pila de ejecución de código síncrono
  • Callback Queue: Cola para callbacks de eventos y timers
  • Microtask Queue: Cola prioritaria para promesas
  • Event Loop: Coordina la ejecución entre las colas

Orden de Ejecución:

  • 1. Código síncrono en el Call Stack
  • 2. Microtasks (Promesas)
  • 3. Macrotasks (setTimeout, setInterval, eventos)

Scope y Closures

El scope determina la accesibilidad de variables y funciones en diferentes partes del código.

Tipos de Scope:

  • Global Scope: Variables accesibles en todo el programa
  • Function Scope: Variables solo accesibles dentro de la función
  • Block Scope: Variables let/const solo accesibles en el bloque
  • Module Scope: Variables accesibles solo en el módulo

Closures:

  • Funciones que recuerdan su scope de creación
  • Permiten encapsulación de datos
  • Útiles para crear datos privados
  • Base para patrones de diseño avanzados
